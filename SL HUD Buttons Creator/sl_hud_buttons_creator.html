🖼️ Texture Templates
This is the "UV layout" - the map that tells SL how a flat image wraps onto the model.

Per button: Each button uses the full image on its own. Drop any square texture on a face and it fills that button perfectly. Best for individual button textures.

Atlas: All buttons share one image, each using its own part of it. Useful when you want one big texture covering all buttons (good with the shared face modes).

Paint Template

Open in Photoshop/GIMP. Paint inside the outlines to make textures that fit your buttons exactly.
⚡ Upload Cost & Detail Levels
"LOD" (level of detail) means SL swaps in simpler versions of a model as you move away from it.
For HUDs: not important - a HUD always sits right on your screen. Just keep curve smoothness low.
For objects you rez in-world: extra detail levels cut Land Impact a lot and keep the shape looking good from far away.
When on, "Download" gives you 4 files (LOWEST/LOW/MED/HIGH). In SL's upload window, load each one into its matching Level of Detail slot. The simpler versions greatly lower the Land Impact of in-world objects.
SL HUD performance tips:
• Keep curve smoothness low - 8 is plenty for circles on a HUD.
• Need outlines on 5 or more buttons? Use the Shared Edge face mode.
• With the per button texture layout, one square texture fits any button shape.
• Skip the neon outline and Per Button mode fits up to 8 buttons in one model.

Export

📖 SL Tips & Workflow

Face limit: SL allows max 8 faces per uploaded model. The counter goes red if you are over. Switch Face Mode to Shared Edge or All Shared to fit more buttons.

Face names: Each face gets a name like mat_btn_1_fill or mat_btn_play_edge. Label your buttons (Per-Button section) so the names tell you which button is which.

Glow setup in SL: Build → Features → Select Face → Glow 0.15-0.3, Full Bright ON.

HUD vs in-world: A HUD never switches detail levels (it is always close). For objects you rez in-world, turn on the extra detail levels for low Land Impact.

Textures: With the default texture layout, any square texture dropped on a face fills that button perfectly.

Click detection in LSL: Use llDetectedTouchFace() - in Per Button mode the face number tells you directly which button was clicked.

Drag preview: Drag inside the preview to live-resize buttons.
